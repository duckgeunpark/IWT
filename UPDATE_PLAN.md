## 🟠 높음 (곧 해야 함)

### 4. 사진 필터링 파이프라인
- **상태**: 예정
- **내용**:
  - **1단계 - 중복 제거** (클라이언트, 비용 없음)
    - 파일 해시(MD5/SHA) 비교 → 완전 동일 파일 제거
    - 파일명 달라도 내용 같으면 제거
  - **2단계 - 연사/버스트 그룹화** (클라이언트, 비용 없음)
    - 촬영 시간 차이 < 수 초 + 같은 GPS → 연사로 판단
    - 그룹 내 대표 1장 자동 선택
  - **3단계 - GPS 없는 사진 분리** (클라이언트, 비용 없음)
    - GPS 데이터 없는 사진 → 경로 활용 불가, 별도 분류
    - 사용자가 수동으로 위치 지정 가능 (기존 기능 활용)
  - **4단계 - 같은 장소 그룹화** (클라이언트, 비용 없음)
    - GPS 좌표 반경 50-100m 이내 + 시간 근접 → 같은 장소로 그룹화
    - 장소당 대표 1장 → 경로용, 나머지 → 장소 앨범
  - **5단계 (선택) - AI 품질 분석** (API 비용 발생)
    - 흐린 사진 감지/제거
    - 구도/품질 점수 매기기
    - 장소별 "제일 잘 나온 사진" 자동 추천
    - 사진 내용 태깅 (음식, 풍경, 인물 등)
  - **6단계 - 쓰레기 데이터 구분**
    - 다른 사진과 아예 관계 없는 사진 제거(활용 안함)
    - 시간대가 맞지 않는 사진 구분 후 제거 
      - ex: 1번사진 2024년 / 2번사진 2018년 등
      - 많은 사진 기반으로 필요없는 것 구분 후 처리(좀더 나은 방안 연구 필요)
  - **7단계 - 데이터 활용 내역 표기**
    - 표기를 숨기던 표시해주던 어떤 사진에대한 활용 내역을 알려줘야 한다 생각
    - 예시
      - 1번사진 : gps및 게시글에 활용됨
      - 2번사진 : 활용하지 않음 ( 내용과 관계 없음 code : 0001(gps) )
      - 3번사진 : 활용하지 않음 ( 시간 대가 맞지 않음 code : 0002(time) )
      - 4번사진 : 사진 기반 llm에 활용됨

### 5. Google Directions API 연동
- **상태**: 예정
- **내용**:
  - 현재 직선(geodesic) 연결 → 실제 도로 기반 경로로 변경
  - 구간별 이동 시간/거리 표시
  - 전체 여행 총 이동 시간/거리 계산
- **비용**: 월 $200 무료 크레딧 (약 40,000건/월 가능)

### 6. 알림 시스템
- **상태**: 예정
- **내용**:
  - 내 게시글에 댓글/좋아요 알림
  - 팔로우한 사람의 새 게시글 알림
  - 추천 코스 알림
- **이유**: 소셜 기능과 함께 사용자 재방문율에 직결

### 7. 데이터 정규화 설계 (AI 학습용)
- **상태**: 예정
- **내용**: 나중에 꺼내 쓰기 편한 구조로 저장
  ```
  trips (여행)
    ├─ routes (경로)
    │   ├─ stops (경유지)
    │   │   ├─ place_id (장소)
    │   │   ├─ arrival_time / departure_time
    │   │   └─ photos[]
    │   └─ segments (구간)
    │       ├─ transport_mode
    │       ├─ duration / distance
    ├─ tags (테마: 맛집, 자연, 문화 등)
    └─ metadata (시즌, 인원, 예산 등)
  ```
- **이유**: 지금부터 잘 쌓아야 나중에 AI 학습/추천에 활용 가능

### 8. 테스트 코드
- **상태**: 예정
- **내용**:
  - 백엔드 API 유닛 테스트
  - 프론트 컴포넌트 테스트

## 🟡 중간 (기능 안정화 후)

### 9. 이동 수단 자동 추정
- **상태**: 예정
- **내용**:
  - 사진 촬영 시간 + 두 지점 간 거리로 평균 속도 계산
  - 속도 기반 이동 수단 매칭:
    - 도보: ~4-5km/h
    - 자전거: ~15-20km/h
    - 차량: ~30-80km/h
    - 대중교통: ~20-40km/h
  - 이동 수단 선택 UI (도보/차량/대중교통/자전거)
  - 자동 추정값을 기본값으로, 사용자가 수정 가능하게

### 10. 자동 게시글 생성 고도화
- **상태**: 예정
- **목표**: 사진 여러 장 업로드 → 프로그램이 자동으로 여행 게시글 생성
- **자동 생성 내용**:
  - 지도 경로 (실제 도로 기반)
  - 장소별 정보 (이름, 유형, 주소)
  - 구간별 이동 수단 + 이동 시간/거리
  - 장소별 추정 체류 시간
  - 장소별 대표 사진 배치
  - 일자별 타임라인 구성
- **사용자 플로우**:
  ```
  사진 30장 업로드
    → [자동] 필터링 (중복/연사/저품질 제거) → 15장
    → [자동] 장소 그룹화 + 경로 생성 → 경로용 8장 + 장소 앨범
    → [자동] 게시글 초안 생성
    → [사용자] 확인/수정 후 게시
  ```

### 11. 경로 추천 시스템
- **상태**: 예정 (데이터 축적 후)
- **내용**:
  - "부산 2박3일" → 실제 사용자 경로 기반 추천
  - 인기 경로 / 숨은 명소 경로 / 최적 동선

  - 시즌별 인기 장소 변화

### 12. 성능 최적화

## 🔵 장기 목표

### 20. 자체 AI 모델 학습
- **상태**: 연구 예정
- **전제 조건**: 충분한 데이터 축적 (최소 수만 건의 여행 데이터)
- **단계별 접근**:
  - **1단계 - 외부 API 활용** (현재~초기)
    - Google Places, Directions API 등으로 기능 구현
    - Claude/GPT API로 게시글 텍스트 생성
    - 이 과정에서 데이터 축적
  - **2단계 - 경량 모델 학습** (데이터 수천~수만 건)
    - 이동 수단 분류 모델 (시간/거리/속도 → 도보/차량/대중교통)
    - 체류 시간 예측 모델 (장소 유형 + 시간대 + 요일)
    - 사진 품질 점수 모델
    - 도구: Python + scikit-learn, TensorFlow Lite 등
  - **3단계 - 추천 모델** (데이터 수만 건 이상)
    - 경로 추천 (사용자 취향 + 유사 사용자 패턴)
    - 장소 추천 (협업 필터링 + 콘텐츠 기반)
    - 개인화 여행 플래너
  - **4단계 - 고도화** (대규모 데이터)
    - 자체 여행 특화 LLM 파인튜닝
    - 사진 → 여행기 자동 생성 모델
    - 실시간 추천 엔진
- **현실적 고려사항**:
  - 초기에는 외부 API로 기능 구현하면서 데이터 쌓는 게 핵심
  - 자체 학습은 데이터 양에 따라 결정
  - 개인정보 처리 동의 / 데이터 익명화 필수

## 로드맵 요약

```
[1단계 - 기반] 🔴 긴급
 소셜 기능 → 검색 강화 → 경로 토글
    ↓
[2단계 - 핵심] 🟠 높음
 사진 필터링 → Directions API → 알림 → 데이터 정규화 → 테스트
    ↓
[3단계 - 고도화] 🟡 중간
 이동수단 추정 → 자동 게시글 → 경로 추천 → 성능/UX 개선
    ↓
[4단계 - 확장] 🟢 낮음
 머문시간 → 통계 → 실시간 → 내보내기 → 플래너 → 개인화
    ↓
[5단계 - AI] 🔵 장기
 자체 모델 학습 → 추천 엔진 → LLM 파인튜닝
```
